# 聊天记录重复问题修复总结

## 🔍 问题现象

用户反馈：
1. 历史记录侧边栏显示很多**重复的记录**（例如：多条相同的"我觉得自己像个反社会女反派"）
2. 聊天记录"不全"（可能被重复记录掩盖）
3. 底部显示了不相关的示例内容（Alex, Sophie, Mike 等）

## 🔧 修复内容

### 1. 修复 ChatHistorySidebar 的重复加载问题

**问题**: 两个 `useEffect` 监听 `isOpen`，导致数据被加载多次

#### 修改前
```typescript
// 第一个 useEffect
useEffect(() => {
  if (isOpen) {
    console.log('🔍 [SIDEBAR] 侧边栏打开，开始加载数据，当前标签:', activeTab)
    loadHistory()
  }
}, [isOpen, activeTab])

// 第二个 useEffect（重复！）
useEffect(() => {
  if (isOpen && activeTab === 'images') {
    console.log('🔍 [SIDEBAR] 确保加载故事记录数据')
    loadHistory()
  }
}, [isOpen])
```

#### 修改后
```typescript
// 只保留一个 useEffect，避免重复加载
useEffect(() => {
  if (isOpen) {
    console.log('🔍 [SIDEBAR] 侧边栏打开，开始加载数据，当前标签:', activeTab)
    loadHistory()
  }
}, [isOpen, activeTab])
```

**效果**: 避免了数据被加载 2 次

### 2. API 层面添加智能去重逻辑

**文件**: `app/api/user/generated-content/route.ts`

**问题**: 数据库中可能存在重复记录（相同的 initialPrompt，短时间内多次保存）

#### 新增功能

##### 2.1 重复记录检测（用于调试）
```typescript
// 🔥 检查是否有重复的 initialPrompt（用于调试）
const promptCounts = new Map<string, number>()
contents.forEach(c => {
  const prompt = c.initialPrompt || ''
  promptCounts.set(prompt, (promptCounts.get(prompt) || 0) + 1)
})
const duplicates = Array.from(promptCounts.entries()).filter(([_, count]) => count > 1)
if (duplicates.length > 0) {
  console.warn('⚠️ [CONTENT-API] 发现重复的 prompt:', duplicates.map(([prompt, count]) => 
    `"${prompt.substring(0, 30)}..." x${count}`
  ))
}
```

**效果**: 在服务器日志中可以看到哪些 prompt 有重复

##### 2.2 智能去重算法
```typescript
// 🆕 去重逻辑：如果 initialPrompt 相同且创建时间接近（5分钟内），只保留最新的一条
const deduplicatedContents = contents.reduce((acc, current) => {
  const existingIndex = acc.findIndex(item => {
    if (!item.initialPrompt || !current.initialPrompt) return false
    const isSamePrompt = item.initialPrompt.trim() === current.initialPrompt.trim()
    const timeDiff = Math.abs(new Date(item.createdAt).getTime() - new Date(current.createdAt).getTime())
    const isNearTime = timeDiff < 5 * 60 * 1000 // 5分钟内
    return isSamePrompt && isNearTime
  })
  
  if (existingIndex === -1) {
    acc.push(current)
  } else {
    // 如果找到相同的，保留较新的那个
    const existingTime = new Date(acc[existingIndex].createdAt).getTime()
    const currentTime = new Date(current.createdAt).getTime()
    if (currentTime > existingTime) {
      acc[existingIndex] = current
    }
  }
  return acc
}, [] as typeof contents)

console.log(`✅ [CONTENT-API] 去重后: ${deduplicatedContents.length}条 (移除了 ${contents.length - deduplicatedContents.length} 条重复)`)
```

**去重规则**:
- ✅ 相同的 `initialPrompt`（去除首尾空格后比较）
- ✅ 创建时间相差不超过 5 分钟
- ✅ 保留较新的记录，移除较旧的

**示例**:
```
原始数据（从数据库查询）:
1. "我觉得自己像个反社会女反派" - 2025/10/31 13:00:00
2. "我觉得自己像个反社会女反派" - 2025/10/31 13:01:30  ← 重复
3. "我觉得自己像个反社会女反派" - 2025/10/31 13:02:45  ← 重复
4. "我就想 我要是没出国留学"    - 2025/10/30 10:00:00
5. "我觉得自己像个反社会女反派" - 2025/10/30 08:00:00  ← 不重复（时间差超过5分钟）

去重后:
1. "我觉得自己像个反社会女反派" - 2025/10/31 13:02:45  ← 保留最新的
2. "我就想 我要是没出国留学"    - 2025/10/30 10:00:00
3. "我觉得自己像个反社会女反派" - 2025/10/30 08:00:00  ← 保留（时间差超过5分钟）
```

## ⚠️ 已知限制

### 1. 不同时间段的相同 prompt 会保留
如果用户在不同时间（相隔超过 5 分钟）输入相同的内容，这些记录会被保留。

**示例**:
```
✅ 保留: "今天上班很开心" - 2025/10/31 09:00
✅ 保留: "今天上班很开心" - 2025/10/31 18:00  ← 相隔 9 小时，不去重
```

**原因**: 这是合理的，因为用户可能真的在不同时间有相同的想法，且生成的图片和故事不同。

### 2. 只是显示层去重
去重只在 API 返回时进行，**数据库中的重复记录仍然存在**。

如果需要清理数据库中的重复记录，需要运行数据库清理脚本（未实现）。

### 3. 0 张图片的记录
某些记录显示"0 张图片"，这可能是因为：
- 图片生成失败
- 保存时 `images` 字段为空
- `imageCount` 为 0

**建议**: 可以在 API 中过滤掉 `imageCount === 0` 的记录：
```typescript
// 在查询后过滤
const filteredContents = contents.filter(c => (c.imageCount || 0) > 0)
```

## 💡 后续优化建议

### 1. 数据库清理脚本
创建一个清理脚本，删除真正的重复记录：

```typescript
// scripts/clean-duplicates.ts
import { prisma } from '@/lib/prisma'

async function cleanDuplicates() {
  // 查找所有用户
  const users = await prisma.user.findMany()
  
  for (const user of users) {
    // 查找该用户的所有内容
    const contents = await prisma.userGeneratedContent.findMany({
      where: { userId: user.id },
      orderBy: { createdAt: 'desc' }
    })
    
    // 找出重复的
    const toDelete: string[] = []
    const seen = new Map<string, any>()
    
    for (const content of contents) {
      const key = `${content.initialPrompt}::${Math.floor(new Date(content.createdAt).getTime() / (5 * 60 * 1000))}`
      
      if (seen.has(key)) {
        // 删除较旧的记录
        toDelete.push(content.id)
      } else {
        seen.set(key, content)
      }
    }
    
    // 批量删除
    if (toDelete.length > 0) {
      await prisma.userGeneratedContent.deleteMany({
        where: { id: { in: toDelete } }
      })
      console.log(`✅ 用户 ${user.email}: 删除了 ${toDelete.length} 条重复记录`)
    }
  }
}
```
