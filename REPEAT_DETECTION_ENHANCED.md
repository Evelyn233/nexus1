# 重复问题检测 - 最终强化版

## 问题回顾

**用户场景：**
```
元键入："我昨天晚上想到了一个产品的idea 很开心 然后早上醒得早早早来了公司"

Q1: "公司在哪个区域？周边环境是什么样的？"
A1: "公司在上海市淞虹路 楼下有商圈 早上来公司时还没人来"
    ↓
Q2: "公司在哪个区域？早上来公司时周围环境怎么样？" ❌ 重复！
    ↓
Q3: "公司具体在哪个区域？周边环境有什么特点？" ❌❌ 又重复！
```

**根本原因：** LLM没有第一时间看到重复检测警告

## 解决方案

### 之前的结构（问题）

```
System Prompt (很长)
    ↓ 
    (重复检测规则埋在中间)
    ↓
User Message
    ↓
LLM可能没仔细看重复检测 ❌
```

### 现在的结构（正确）

```
User Message 开头：
🚨🚨🚨 【重复检测 - 最高优先级】🚨🚨🚨

用户之前的回答：公司在淞虹路 楼下有商圈

强制检查：
❌ 地点已回答！禁止再问：公司在哪、哪个区域
❌ 环境已回答！禁止再问：周边环境、环境怎样

如果上面有任何警告，必须返回 {"questions": []}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    ↓
LLM第一时间看到警告 ✅
    ↓
返回 {"questions": []} ✅
```

## 检测逻辑

### 地点检测

```javascript
if (allAnswers.includes('路') || 
    allAnswers.includes('区') || 
    allAnswers.includes('街') || 
    allAnswers.includes('淞虹路')) {
  警告：'❌ 地点已回答！禁止再问地点'
}
```

### 环境检测

```javascript
if (allAnswers.includes('商圈') || 
    allAnswers.includes('楼下') || 
    allAnswers.includes('周边') || 
    allAnswers.includes('环境')) {
  警告：'❌ 环境已回答！禁止再问环境'
}
```

### 次数检测

```javascript
if (previousAnswers.length >= 3) {
  警告：'❌ 已经问了3个问题！必须返回空数组'
}
```

## 预期效果

### 现在的对话流程

```
用户："我想到idea很开心 来了公司"

Q1: "公司在哪个区域？"
A1: "公司在淞虹路 楼下有商圈"
    ↓
【重复检测】
✅ 检测到"淞虹路" → 地点已回答
✅ 检测到"商圈" → 环境已回答
    ↓
返回 {"questions": []}
    ↓
直接跳转生图 ✅
```

不会再有Q2、Q3重复问地点！

## 关键改进

### 1. ✅ 警告前置

重复检测警告放在用户消息的**最前面**，LLM第一眼就看到。

### 2. ✅ 简化逻辑

```
之前：复杂的分类和案例（太长）
现在：直接检测关键词 + 简短警告
```

### 3. ✅ 强制指令

```
"如果上面有任何警告，必须返回 {"questions": []}"
```

明确告诉LLM该怎么做。

### 4. ✅ 视觉分隔

```
🚨🚨🚨 【重复检测】 🚨🚨🚨
━━━━━━━━━━━━━━━━━━━━━━
```

用emoji和分隔线吸引注意力。

## 检测触发词

| 类别 | 触发词 | 禁止再问 |
|------|-------|---------|
| 地点 | 路/区/街/淞虹路/朝阳区 | 公司在哪、哪个区域、具体在哪 |
| 环境 | 商圈/楼下/周边/环境/附近 | 周边环境、环境怎样、有什么特点 |
| 次数 | ≥3个回答 | 任何问题 |

## 总结

✅ **重复检测前置** - LLM第一眼看到
✅ **关键词检测** - 简单直接
✅ **强制返回空** - 明确指令
✅ **视觉突出** - emoji + 分隔线

**现在LLM应该不会再重复问地点和环境了！** 🎯✨
