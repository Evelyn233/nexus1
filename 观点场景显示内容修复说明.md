# 观点场景显示内容修复说明

## 🐛 问题描述

**用户反馈**：观点场景显示的是图片生成提示词，而不是观点描述。

**示例**：
用户看到：
```
高质量杂志插图风格的概念场景:一个巨大的工业时钟机械装置在灰色工厂环境中运转,齿轮和传送带象征着僵化的社会体系。在机械阴影中,一个26岁中国女性观察者（165cm,长发）冷静地站在边缘,手中握着一颗微光闪烁的种子,象征被压抑的创造力和潜在的反抗。机械装置吞噬着色彩斑斓的创意碎片,而观察者的目光穿透机械结构的缝隙,寻找体系的弱点
```

**问题**：这是 `imagePrompt`（图片生成提示词），不是给用户看的观点描述。

## 🔍 问题分析

### 1. 观点场景数据结构

观点场景生成时，LLM返回的JSON包含：

| 字段 | 用途 | 示例内容 |
|------|------|----------|
| `opinion` | 观点文本 | "社会结构僵硬机械" |
| `opinionType` | 观点类型 | "社会现象评论" |
| `sceneDescription_CN` | **技术性场景描述** | "详细描述主角群体的行为，展现观点现象" |
| `imagePrompt` | **图片生成提示词** | "高质量杂志插图风格的概念场景..." ❌ 技术性 |
| `storyFragment` | **人文叙事** | "观点叙事（100-150字），体现观点的社会意义和人文价值" ✅ 用户友好 |

### 2. 前端显示逻辑

#### 问题位置：`app/chat-new/page.tsx` 第1659行

**修复前**：
```typescript
storyFragment: scene.sceneDescription_CN || scene.description_zh || scene.storyFragment || ''
//             ^^^^^^^^^^^^^^^^^^^^^^^^ 
//             优先使用了技术性描述！
```

这导致显示优先级：
1. ❌ `sceneDescription_CN` - 技术性场景描述
2. ❌ `description_zh` - 可能不存在
3. ✅ `storyFragment` - 人文叙事（但排在最后！）

**修复后**：
```typescript
storyFragment: scene.storyFragment || ''
//             ^^^^^^^^^^^^^^^^^^^^
//             只使用人文叙事！
```

### 3. 为什么会出现这个问题？

**设计初衷**：
- `sceneDescription_CN` 是给内部使用的，描述场景的技术细节
- `storyFragment` 是给用户看的，包含人文叙事和观点意义
- `imagePrompt` 是给AI生图用的，包含所有技术参数

**实际情况**：
前端错误地优先使用了 `sceneDescription_CN`，导致技术性内容显示给用户。

## 🔧 修复方案

### 修复代码

**文件**：`app/chat-new/page.tsx`

**位置**：第1659行

**修改**：
```diff
  const opinionScene = {
    ...scene,
    isOpinionScene: true,
    imagePrompt: scene.imagePrompt,
    opinionText: scene.opinionText || scene.opinion || scene.title?.replace('观点：', ''),
-   storyFragment: scene.sceneDescription_CN || scene.description_zh || scene.storyFragment || ''
+   storyFragment: scene.storyFragment || ''  // 🔥 只使用storyFragment（人文叙事）
  }
```

### 为什么这样修复？

1. **明确字段用途**：
   - `storyFragment` = 人文叙事（给用户看）
   - `sceneDescription_CN` = 技术描述（内部使用）
   - `imagePrompt` = 生图提示词（给AI用）

2. **保持一致性**：
   - 观点场景的 `storyFragment` 专门设计为"观点叙事...体现观点的社会意义和人文价值"
   - 应该只使用这个字段，而不是fallback到技术性字段

3. **用户体验**：
   - 用户应该看到的是易读的人文叙事
   - 而不是技术性的场景描述或图片生成参数

## 🎯 相关场景类型对比

| 场景类型 | 使用字段 | 原因 |
|---------|---------|------|
| 基础场景 | `storyFragment` | 故事叙事 |
| 心理剧 | 拼接的心理分析文字 | 情绪深度 |
| 假想场景 | `sceneDescription_CN` ✅ | 假想场景的 `sceneDescription_CN` **专门设计**为叙事铺垫 |
| 观点场景 | `storyFragment` ✅ | **不能用** `sceneDescription_CN`（技术性） |

### 为什么假想场景可以用 `sceneDescription_CN`？

**假想场景的 LLM Prompt 要求**：
```
sceneDescription_CN（叙事铺垫）：
- ✅ 必须在开头明确这是"假想/幻想/平行人生"
- ✅ 使用铺垫语句："在她想象的平行人生中..."、"这是她无数次幻想过的画面：..."
```

假想场景的 `sceneDescription_CN` 本身就是设计为**叙事内容**，而不是技术描述。
