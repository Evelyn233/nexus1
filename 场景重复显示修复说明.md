# 场景重复显示修复说明

## 🐛 问题描述

**用户反馈**：现在有好多重复的场景显示。

**症状**：
- 同一个场景（如"假想:社会结构摧毁者的天台观察"）显示了多次
- 同一个观点场景（如"只关心对错不关心死活"）显示了多次
- 控制台显示多个相同的场景卡片

## 🔍 问题分析

### 1. 根本原因

**占位消息ID冲突**：

前端有两个地方生成图片：
1. **基础场景生成**（1738行）：使用 `image-placeholder-${i}`
2. **额外场景生成**（1180行）：使用 `image-placeholder-${globalIndex}`

额外场景分3批调用 `generateImagesForNewScenes`，每次都传入 `startIndex=0`：
```typescript
// 第1批：心理剧（1613行）
generateImagesForNewScenes([psychodramaScene], 0, ...)  
// → placeholderId = `image-placeholder-0` ❌

// 第2批：假想场景（1645行）
generateImagesForNewScenes(processedHypotheticalScenes, 0, ...)
// → placeholderId = `image-placeholder-0`, `image-placeholder-1`, ... ❌ 重复！

// 第3批：观点场景（1674行）
generateImagesForNewScenes(processedOpinionScenes, 0, ...)
// → placeholderId = `image-placeholder-0`, `image-placeholder-1`, ... ❌ 重复！
```

### 2. 冲突示例

| 批次 | 场景索引 | 占位ID | 冲突？ |
|------|----------|--------|--------|
| 心理剧 | 0 | `image-placeholder-0` | ❌ 首次 |
| 假想场景1 | 0 | `image-placeholder-0` | ❌ 重复！|
| 假想场景2 | 1 | `image-placeholder-1` | ❌ 首次 |
| 假想场景3 | 2 | `image-placeholder-2` | ❌ 首次 |
| 观点场景1 | 0 | `image-placeholder-0` | ❌ 重复3次！|
| 观点场景2 | 1 | `image-placeholder-1` | ❌ 重复！|
| 观点场景3 | 2 | `image-placeholder-2` | ❌ 重复！|
| 观点场景4 | 3 | `image-placeholder-3` | ✅ 首次 |

### 3. React行为

当执行 `setMessages(prev => prev.map(msg => msg.id === placeholderId ? ...))` 时：
- React会查找所有 `id === 'image-placeholder-0'` 的消息
- 如果有多个相同ID的占位消息，都会被替换为同一个场景
- 导致同一个场景显示多次

## 🔧 修复方案

### 使用唯一的占位ID

**修复前**：
```typescript
// ❌ 容易冲突
const placeholderId = `image-placeholder-${globalIndex}`
```

**修复后**：
```typescript
// ✅ 唯一ID：索引 + 时间戳 + 随机数
const placeholderId = `image-placeholder-${globalIndex}-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`
```

### 修复位置

#### 1. 额外场景生成（1180行）
```typescript
// 🔥 添加占位消息（使用唯一ID，避免不同批次的场景ID冲突）
const placeholderId = `image-placeholder-${globalIndex}-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`
setMessages(prev => [...prev, {
  id: placeholderId,
  type: 'system',
  content: `⚡ 正在创作第 ${globalIndex + 1} 幅作品...`
}])
```

#### 2. 基础场景生成（1738行）
```typescript
// 🔥 添加生成中的占位消息（使用唯一ID）
const placeholderId = `base-image-placeholder-${i}-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`
setMessages(prev => [...prev, {
  id: placeholderId,
  type: 'system',
  content: `⚡ 正在创作第 ${i + 1} 幅作品...`
}])
```

## 🎯 技术细节

### ID生成策略

```typescript
`${prefix}-${index}-${timestamp}-${random}`
```

- **prefix**：场景类型前缀（`base-image-placeholder` 或 `image-placeholder`）
- **index**：场景索引（0, 1, 2, ...）
- **timestamp**：当前时间戳（`Date.now()`）
- **random**：9位随机字符串（`Math.random().toString(36).substr(2, 9)`）

### 示例ID
```
base-image-placeholder-0-1730362400000-k5j8h3g2f
image-placeholder-1-1730362401234-m9n7v4b2c
image-placeholder-0-1730362401567-p3q6r8t1x
```

### 为什么这样设计？

1. **时间戳**：确保不同时间创建的占位消息ID不同
2. **随机数**：处理极端情况（同一毫秒内创建多个占位消息）
3. **索引**：保留语义，方便调试
4. **前缀**：区分基础场景和额外场景
