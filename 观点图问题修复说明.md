# 观点图问题修复说明

## 问题概述

1. **观点图重复问题**：前端返回的观点图出现重复
2. **图片无法返回问题**：部分图片无法正确返回

## 修复内容

### 1. 观点图重复问题修复 (`lib/opinionVisualizationService.ts`)

#### 修复前的问题：
- 观点去重逻辑不够完善，只有单层相似度检测
- 相似度阈值设置过低（50%），导致相似观点未被过滤
- 没有精确文本匹配的第一层去重，完全相同的观点可能被重复生成
- LLM提示词限制"最多返回2个观点"，导致有效观点可能被遗漏

#### 修复后的改进：

**A. 增强去重逻辑**（第52-86行）
```typescript
// 🔥 新增两层去重机制
1. 第一层：精确文本匹配去重
   - 使用 Set<string> 记录已见过的观点文本
   - 将观点文本标准化（转小写、去空格）后比较
   
2. 第二层：相似度检测去重
   - 将相似度阈值从 50% 提升到 60%（更严格）
   - 只有相似度低于60%的观点才会被保留
```

**B. 优化观点场景生成策略**（第976-979行）
```typescript
// 🔥 生成所有不重复的观点场景（不限制数量）
// 通过强化的去重机制确保每个观点都是独特的
const selectedOpinions = uniqueOpinions // 使用所有去重后的观点
```

**C. 增加详细日志**
```typescript
// 每一步都有详细的日志输出
console.log(`✅ [OPINION] 添加唯一观点: "${opinion.text}" (优先级: ${opinion.priority})`)
console.log(`🔄 [OPINION] 跳过相似观点: "${opinion.text}"`)
console.log(`📋 [OPINION] 保留的观点:`, uniqueOpinions.map(o => o.text))
console.log(`✅ [OPINION] 将为 ${selectedOpinions.length} 个不重复的观点生成场景`)
```

**D. 优化LLM提示词**（第300行）
```typescript
// 从"最多返回2个观点"改为"返回所有检测到的观点"
// 让LLM返回所有观点，由去重机制保证不重复
- 返回所有检测到的观点（会通过去重机制避免重复）
```

### 2. 图片无法返回问题修复 (`app/api/seedream-generate/route.ts`)

#### 修复前的问题：
- 只支持两种API响应格式（data.data 和 data.url）
- 如果API返回格式不符合预期，图片就无法提取
- 每次请求生成4张图（n: 4），可能导致资源浪费和重复

#### 修复后的改进：

**A. 支持多种API响应格式**（第94-150行）
```typescript
// 🔥 新增支持4种API响应格式
1. data.data 数组格式（标准格式）
2. 直接包含 url 字段
3. choices 格式（某些API返回）
4. images 数组格式

// 每种格式都有对应的处理逻辑和日志输出
```

**B. 完善错误提示**
```typescript
// 如果所有格式都无法提取图片URL
if (imageUrls.length === 0) {
  console.error('❌ [API] 未能从响应中提取到图片URL')
  console.error('📦 [API] 响应数据结构:', Object.keys(data))
  console.error('📦 [API] 完整响应:', JSON.stringify(data, null, 2))
  // 返回详细错误信息
}
```

**C. 优化生图参数**（第16-39行）
```typescript
// 🔥 从 n: 4 改为 n: 1
// 🔥 从 max_images: 4 改为 max_images: 1
// 每个场景只生成1张图，避免重复和资源浪费

const requestBody = {
  ...
  sequential_image_generation_options: {
    max_images: 1 // 改为1
  },
  n: 1 // 改为1
}
```

**D. 增加详细请求日志**
```typescript
console.log('📋 [API] 请求参数:', {
  model: requestBody.model,
  promptLength: prompt.length,
  size: requestBody.size,
  n: requestBody.n,
  max_images: requestBody.sequential_image_generation_options.max_images
})
```
