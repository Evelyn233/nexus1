# 用户信息收集改进

## 改进内容

### 1. ✅ 添加用户称呼输入
- **新增**：姓名/称呼输入框（第一个问题）
- **位置**：用户信息页面最顶部
- **必填**：姓名现在是必填字段

### 2. ✅ 一次输入，永久保存
- 用户完成信息填写后，信息会保存到 localStorage
- 下次访问时，会自动检查是否已有完整信息
- 如果信息完整，直接跳转到主页，**不再重复询问**

### 3. ✅ 个性化体验
- 在后续问题中，会使用用户的称呼
- 例如："张三，请告诉我您的性别？"

## 用户体验流程

### 首次访问
```
1. 访问网站
   ↓
2. 检测到无用户信息
   ↓
3. 跳转到 /user-info 页面
   ↓
4. 填写信息（包括称呼）
   ↓
5. 信息保存到 localStorage
   ↓
6. 跳转到 /home 主页
```

### 再次访问
```
1. 访问网站
   ↓
2. 检测到已有完整用户信息
   ↓
3. 直接跳转到 /home 主页 ✅
```

## 信息完整性检查

用户信息需要包含以下字段才算完整：

### 必填字段（所有用户）
- ✅ **姓名/称呼**（新增）
- ✅ 性别
- ✅ 出生年月日
- ✅ 身高
- ✅ 体重
- ✅ 所在地
- ✅ 性格描述

### 额外字段（女性用户）
- ✅ 头发长度

## 如何重新填写信息

如果用户想修改信息：

1. 在主页点击右上角的"👤"图标
2. 查看当前信息
3. 点击"编辑信息"按钮
4. 跳转到用户信息页面重新填写

或者：

1. 在主页点击"重新设置用户"按钮
2. 清空所有信息
3. 重新填写

## 用户数据管理

### 查看用户信息
- 主页右上角 "👤" 按钮
- 显示当前保存的所有信息

### 编辑用户信息
- 主页 → 个人信息弹窗 → "编辑信息" 按钮
- 跳转到 /user-info 页面
- 已有信息会预填充

### 清空用户信息
- 主页 → 个人信息弹窗 → "重新设置用户" 按钮
- 确认后清空所有本地数据
- 下次访问需要重新填写

## 注意事项

1. **隐私保护**
   - 所有数据存储在用户浏览器本地
   - 不会上传到服务器
   - 清除浏览器缓存会丢失数据

2. **数据同步**
   - 如果使用 Memobase，会同步到云端
   - 清除本地缓存后可从 Memobase 恢复

3. **跨设备使用**
   - 不同设备需要分别填写
   - 除非启用 Memobase 同步功能
