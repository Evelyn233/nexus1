# 首次录入信息等待提示框

## 功能概述

当新用户首次录入个人信息时，系统会显示一个精美的等待提示框，告知用户系统正在处理他们的信息，请稍等片刻。

## 用户体验

### 首次用户流程

```
1. 新用户注册账号
   ↓
2. 跳转到 /user-info 填写详细信息
   ↓
3. 填写完成，点击"完成"按钮
   ↓
4. ✨ 显示首次录入信息等待提示框
   ↓
5. 提示框显示4个步骤：
   - 正在分析您的信息...
   - 生成个性化档案...
   - 建立记忆系统...
   - 完成初始化...
   ↓
6. 每个步骤约1.2秒，总时长约6.5秒
   ↓
7. 自动关闭并跳转到首页
```

### 老用户流程

```
1. 老用户登录
   ↓
2. 如需修改信息，进入 /user-info
   ↓
3. 修改信息，点击"完成"
   ↓
4. ⚡ 直接保存，不显示等待提示框
   ↓
5. 立即跳转到首页
```

## 实现细节

### 组件结构

#### 1. FirstTimeSetupModal 组件
- **位置**: `components/FirstTimeSetupModal.tsx`
- **功能**: 显示等待提示框，展示初始化步骤
- **特点**:
  - 渐进式进度条
  - 4个步骤，逐步展示
  - 自动关闭
  - 精美的UI设计

#### 2. 用户信息页面集成
- **位置**: `app/user-info/page.tsx`
- **修改**:
  - 添加 `isFirstTimeUser` 状态检测
  - 添加 `showFirstTimeModal` 控制提示框显示
  - 在提交时检测是否首次用户
  - 首次用户：显示提示框并延迟跳转
  - 老用户：直接保存并跳转

### 首次用户检测逻辑

```typescript
// 检测是否是首次用户
const hasExistingCompleteInfo = savedInfo.name && 
                               savedInfo.gender && 
                               savedInfo.birthDate?.year && 
                               isInfoComplete
setIsFirstTimeUser(!hasExistingCompleteInfo)
```

**判断标准**:
- ✅ 有姓名
- ✅ 有性别
- ✅ 有出生年份
- ✅ 信息完整

如果以上任一条件不满足，则视为首次用户。

### 时间配置

| 项目 | 时间 | 说明 |
|------|------|------|
| 每步时长 | 1.2秒 | 步骤切换间隔 |
| 步骤数量 | 4步 | 总共4个步骤 |
| 完成延迟 | 1.5秒 | 最后一步显示时间 |
| 跳转延迟 | 6.5秒 | 等待提示框完成后跳转 |

### 提示框步骤

1. **正在分析您的信息**
   - 图标: 蓝色加载动画
   - 说明: AI正在深度分析您的性格特征和偏好

2. **生成个性化档案**
   - 图标: 紫色加载动画
   - 说明: 基于您的信息创建专属用户档案

3. **建立记忆系统**
   - 图标: 绿色加载动画
   - 说明: 为您建立智能记忆和学习系统

4. **完成初始化**
   - 图标: 绿色对勾
   - 说明: 所有设置已完成，即将为您开启个性化体验

## UI设计

### 视觉元素

- **背景**: 半透明黑色蒙层 (bg-black bg-opacity-50)
- **主容器**: 白色圆角卡片，带阴影
- **标题图标**: 紫粉色渐变圆形，中心✨图标
- **进度条**: 紫粉色渐变，平滑过渡
- **步骤卡片**: 
  - 当前步骤: 蓝色背景
  - 已完成: 绿色背景
  - 未开始: 灰色背景

### 响应式设计

- 移动端: max-w-md，适配手机屏幕
- 桌面端: 居中显示，最大宽度限制
- 自适应: 根据屏幕大小调整布局
