# 两层数据架构：原始数据 + 分析层

## 核心理念

**数据库应该是用户日常习惯的记录地方，分为两层：**

1. **第一层：原始数据** - 用户真实说了什么、做了什么
2. **第二层：分析层** - 基于第一层分析出来的，也要存储，用于以后的推测

## 为什么第二层也要存储？

### ❌ 不存储的问题
- 每次都要重新分析，效率低
- 无法追踪分析结果的变化
- 无法用于快速推测

### ✅ 存储的优势
- 分析一次，多次使用
- 可以追踪分析结果的演变
- 快速用于场景生成和问题生成
- 可以重新分析更新（AI模型升级时）

## 数据流程

### 1. 用户对话后

```javascript
用户对话
    ↓
存储到第一层（ChatSession, DailyActivity, UserStatement）
    ↓
触发AI分析
    ↓
更新第二层（UserAnalysis, BehaviorPattern, PreferenceSummary）
```

### 2. 场景生成时

```javascript
获取两层数据
    ↓
构建提示词（优先级：当前输入 > 第一层 > 第二层）
    ↓
生成场景
```

## 关键区别

| 维度 | 第一层：原始数据 | 第二层：分析层 |
|------|----------------|---------------|
| **内容** | 用户说的原话 | AI分析结果 |
| **示例** | "我不喜欢线上开会" | "工作偏好：线下 > 线上" |
| **来源** | 用户直接输入 | 基于第一层AI分析 |
| **可追溯** | 是最终事实 | 指向第一层证据 |
| **是否存储** | ✅ 必须存储 | ✅ 也要存储 |
| **更新频率** | 每次对话后 | 异步分析后 |
| **用途** | 提供事实依据 | 用于推测和生成 |
| **优先级** | 高（事实） | 低（推测） |

## 实际例子

### 场景1：用户说"我累了"

**第一层存储：**
```json
DailyActivity {
  "activity": "线上开会",
  "userState": "累了",
  "userQuote": "线上开会，有点累"
}
```

**第二层分析（基于第一层）：**
```json
PreferenceSummary {
  "preference": "线上开会时容易疲惫",
  "evidence": [{"type": "activity", "id": "act001"}]
}
```

**场景生成时使用：**
```
优先级1：用户当前输入
优先级2：第一层 - 用户说"累了" → 必须体现疲惫感
优先级3：第二层 - 推测：累了可能想独处休息
```

### 场景2：用户说"吃了炒牛肉"

**第一层存储：**
```json
DailyActivity {
  "activity": "吃了云海肴的炒牛肉",
  "userQuote": "今天中午吃了云海肴的炒牛肉"
}
```

**第二层分析（多次后）：**
```json
BehaviorPattern {
  "description": "喜欢吃云南菜",
  "frequency": 3,
  "evidence": [
    {"type": "activity", "id": "act001", "excerpt": "吃了云海肴的炒牛肉"},
    {"type": "activity", "id": "act005", "excerpt": "吃了云南菜"}
  ]
}
```

**场景生成时使用：**
```
优先级1：用户当前输入 "吃了炒牛肉"
优先级2：第一层 - "炒牛肉"必须100%还原（不能改成"柠檬鸡"）
优先级3：第二层 - 推测：可能喜欢云南菜（用于补充其他场景）
```

## 优先级原则

### 场景生成时的优先级

```
用户当前输入："今天吃了炒牛肉，线上开会，累了"
    ↓
优先级1：当前输入（100%还原）
  - 炒牛肉（必须写"炒牛肉"）
  - 线上开会（必须写"线上"）
  - 累了（必须体现疲惫感）
    ↓
优先级2：第一层（事实）
  - 最近活动：用户经常说累（要考虑）
  - 最近陈述："我不喜欢线上开会"（要考虑）
    ↓
优先级3：第二层（推测）
  - 性格分析：INTJ（用于推测行为）
  - 行为模式：喜欢独处（用于推测场景）
  - 偏好总结：线下 > 线上（用于理解用户）
```

### ⚠️ 关键规则

1. **第一层不能被第二层覆盖**
   - 用户说"炒牛肉" → 不能因为第二层推测"喜欢云南菜"就改成"米线"
   
2. **第二层不能违背第一层**
   - 第一层：用户说"我不喜欢线上开会"
   - 第二层：不能分析出"喜欢线上开会"
   
3. **第二层基于第一层**
   - 第二层的每个分析都要指向第一层的证据
   - 可以追溯到具体的对话或活动

## 总结

**两层架构的核心：**

- **第一层**：用户输入的记录（事实，最高优先级）
- **第二层**：AI分析的存储（推测，用于以后的推测和生成）
- **优先级**：当前输入 > 第一层 > 第二层
- **可追溯**：第二层的每个分析都指向第一层的证据
- **双层存储**：两层都要存储，但用途不同

这样既能记录用户的日常习惯（第一层），又能利用AI分析提高推测质量（第二层），还能保证优先级正确！
