# 🎯 用户数据优先级设计

## 核心原则

### ⭐ 第一优先级：用户自我认知 + 真实对话
```
用户说什么 = 最重要的数据来源

例如：
用户填写："INTJ AI创业者 内心文艺 向往硅谷"
  ↓
直接提取：
- MBTI: INTJ
- 关键词: AI, 创业, 文艺, 硅谷
- 特质: 技术导向、创业精神、艺术气质、国际视野

用户对话："昨天晚上没睡着困死了"
  ↓
直接提取：
- 睡眠问题
- 可能的压力源
```

### 📊 第二优先级：八字、星盘（辅助解释）
```
命理信息 = 解释视角，不是数据来源

例如：
用户说："我容易失眠"
  +
八字：丁火日主（内心活跃）
星座：双鱼座（思虑多）
  ↓
AI推理："双鱼座+丁火特征可能让你思虑较多，建议..."

⚠️ 不能反过来：
不能因为是双鱼座就假设用户一定失眠 ❌
```

## 🚫 禁止硬编码

### ❌ 错误做法
```typescript
// 硬编码特质（不管用户实际情况）
coreTraits: [
  '创业精神强烈',  // ❌ 用户可能不创业
  '风险承受能力强', // ❌ 用户可能很保守
  '领导魅力',      // ❌ 用户可能是技术人员
]
```

### ✅ 正确做法
```typescript
// 从用户自我描述提取
personality: "INTJ AI创业者 内心文艺"
  ↓
提取关键词: ["INTJ", "AI", "创业", "文艺"]
  ↓
生成特质:
- "AI" → ["技术导向", "创新思维"]
- "创业" → ["创业精神", "风险承担"]
- "文艺" → ["艺术气质", "审美能力"]
  ↓
coreTraits: [
  "技术导向", "创新思维",  // 基于"AI"
  "创业精神", "风险承担",  // 基于"创业"
  "艺术气质", "审美能力"   // 基于"文艺"
]
```

## 📋 三层数据架构（修正版）

### 第1层：用户自我认知（100%权重）
```
来源：用户填写
内容：
- 姓名、性别、生日
- 自我性格描述："INTJ AI创业者 内心文艺 向往硅谷"
- MBTI类型

处理：
- 直接提取关键词
- 不做推测
- 不添加内容

存储：
users表 → selfMBTI, selfTraits, personality
```

### 第2层：聊天记录（100%权重）
```
来源：用户每次回答
内容：
- Q: "今天什么让你疲惫？"
- A: "昨天晚上没睡着困死了"

处理：
- 直接提取：睡眠问题、工作压力
- 基于回答内容分析
- 不添加没说的内容

存储：
chat_sessions表 → messages, answers
user_metadata表 → conversationInsights（累积）
```

### 第3层：命理分析（辅助解释，20%权重）
```
来源：基于生日计算
内容：
- 星座：双鱼座
- 生肖：兔
- 八字：丁火日主

用途：
- 提供解释视角
- 辅助理解用户行为
- 生成建议时参考

⚠️ 注意：
- 不能凭空推断特质
- 只在用户有相关表现时才引用
- 作为"为什么"的解释，不是"是什么"的判断

例如：
用户说"失眠" → 可以说"双鱼座特质可能让你思虑多" ✅
用户没说失眠 → 不能说"双鱼座所以你应该失眠" ❌
```

## 🔄 正确的分析流程

### 示例：分析用户

```
用户输入：
- 自我认知："INTJ AI创业者 内心文艺 向往硅谷"
- 聊天记录1："我今天上班好累"
- 聊天记录2："昨天晚上没睡着困死了"
- 聊天记录3："在想新产品的设计"

第1步：提取直接信息（第一优先级）
从自我认知：
  ✅ MBTI: INTJ
  ✅ 职业: AI创业
  ✅ 特质: 文艺气质、硅谷梦想

从对话记录：
  ✅ 工作状态: 疲惫
  ✅ 睡眠问题: 失眠
  ✅ 思考内容: 产品设计

第2步：计算命理（辅助）
生日 1999-03-16：
  星座: 双鱼座 → 敏感、思虑多
  生肖: 兔 → 温和、追求和谐
  八字: (需AI分析)

第3步：综合分析（命理辅助解释）
核心特质（100%来自用户）：
  - "技术导向"（来自"AI"）
  - "创业精神"（来自"创业者"）
  - "艺术气质"（来自"内心文艺"）
  - "睡眠质量欠佳"（来自"失眠"）
  - "工作投入度高"（来自"在想产品设计"）

命理解释（辅助理解）：
  - 失眠 + 双鱼座敏感 + 丁火多思 
    → "你可能因思虑过多导致失眠，建议睡前冥想"
  - INTJ + 创业 → "你的战略思维适合产品规划"

⚠️ 不添加用户没说的内容！
```
