# 额外场景流式返回修复说明

## 🐛 问题描述

**用户反馈**：生成完全没有立刻返回给前端，前端只有基础场景的图。

**症状**：
- 后端日志显示假想场景、观点场景、心理剧已生成完成
- 前端只显示了3张基础场景的图片
- 额外生成的场景（假想、观点、心理剧）没有返回给前端

## 🔍 问题分析

### 1. 日志分析

**后端日志（正常）**：
```
✅ [ADDITIONAL-GEN] 假想场景已存储到results，共 3 个场景
✅ [ADDITIONAL-GEN] 后续生成完成
```

**前端日志（缺失）**：
```
❌ 没有看到：✅ [CHAT-NEW] 后台生成完成，收到结果!
❌ 没有看到：🎬 [CHAT-NEW] 假想场景准备生图
❌ 没有看到：🎯 [CHAT-NEW] 观点场景准备生图
❌ 没有看到：🎭 [CHAT-NEW] 心理剧场景准备生图
```

### 2. 根本原因

**Promise错误处理缺失**：

在 `lib/contentGenerationService.ts` 中，有3个并行的Promise：
1. 观点检测和生成
2. 情绪检测和心理剧生成  ✅ 有 `.catch()`
3. 假想场景检测和生成  ❌ 缺少 `.catch()`

如果某个Promise抛出异常且没有 `.catch()` 捕获，会导致：
- `Promise.all(promises)` 失败
- 整个 `generateAdditionalContent` 函数卡住或reject
- 前端的 `.then()` 回调永远不会被触发

## 🎯 技术细节

### Promise.all 的行为

**修复前**：
```typescript
promises.push(
  detectPromise.then(async () => { ... })  // ❌ 没有 .catch()
)
await Promise.all(promises)  // 如果某个promise reject，整个失败
```

**修复后**：
```typescript
promises.push(
  detectPromise.then(async () => { ... })
    .catch(error => { console.error(...) })  // ✅ 捕获错误，不影响其他
)
await Promise.all(promises)  // 所有promise都会完成（即使有错误）
```
