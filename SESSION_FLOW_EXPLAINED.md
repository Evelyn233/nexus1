# 对话流（Session）完整说明

## ✅ 是的！每一次开启聊天都是用一个独立的对话流

## 具体实现

### 代码实现（`app/chat-new/page.tsx:50`）

```typescript
const [currentSessionId, setCurrentSessionId] = useState<string>(`session_${Date.now()}`)
```

每次打开聊天页面，就创建一个新的 `sessionId`，使用时间戳确保唯一性。

## 关键点

### 1. 每次开启 = 新的SessionId

```typescript
// 页面加载时创建
session_1744612345678  // 第1次聊天
session_1744615678901  // 第2次聊天
session_1744618234567  // 第3次聊天
```

### 2. 单个Session内 = 完整上下文

```javascript
session_1744612345678 包含：
{
  sessionId: "session_1744612345678",
  messages: [
    { role: 'user', content: '我高考失利了' },
    { role: 'assistant', content: '当时你的心情如何？' },
    { role: 'user', content: '很难过' },
    { role: 'assistant', content: '后来怎么处理的？' },

  answers: ["很难过", "出国了"],
  questions: ["当时你的心情如何？", "后来怎么处理的？"]

```

### 3. 自动保存（`app/chat-new/page.tsx:173-179`）

```typescript
useEffect(() => {
  if (messages.length > 0) {
    const timer = setTimeout(() => {
      saveSession()  // 自动保存
    }, 1000)
    
    return () => clearTimeout(timer)
  }
}, [messages])  // messages变化就保存
```

## 对话流的生命周期

```
┌─────────────────────────────────────────────┐
│ 1. 首页输入 → 创建新Session                    │
│    sessionId = `session_${Date.now()}`      │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ 2. 对话进行中                                │
│    - User输入 → 添加到messages[]            │
│    - AI回复 → 添加到messages[]              │
│    - 所有对话都在同一个sessionId中           │
│    - 完整的上下文记忆 ✅                     │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ 3. 自动保存                                  │
│    - 每次messages变化后1秒自动保存           │
│    - 保存到数据库（ChatSession表）           │
│    - 使用同一个sessionId                     │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ 4. 对话结束                                  │
│    - 生成内容（场景/心理剧/封面）             │
│    - 分析用户数据（两层数据）                 │
│    - 完整session保存在数据库                 │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ 5. 下次开启新聊天                            │
│    → 创建新的sessionId                       │
│    → 开始新的对话流                          │
└─────────────────────────────────────────────┘
```

## 对话流的特点

### ✅ 优点

1. **完整的上下文记忆**
   - 单个session内所有messages累积
   - AI记得之前所有对话

2. **独立的对话主题**
   - 每次新聊天是独立的主题
   - 不会混淆不同话题

3. **永久保存**
   - 每个session都保存在数据库
   - 可以随时查看历史对话

4. **多设备同步**
   - 基于userId存储
   - 任何设备登录都能看到历史

## 实际示例

### 场景1：讨论高考经历

```
Session: session_1744612345678
────────────────────────────────
User: "我高考失利了"
AI: "当时你的心情如何？"
User: "很难过"           ← AI记得"高考失利"
AI: "后来怎么处理的？"    ← AI记得"高考失利+很难过"
User: "出国了"           ← AI记得完整故事线
AI: "出国后心情好些了吗？" ← AI记得所有之前内容
```

### 场景2：（新的session）寻找咖啡馆

```
Session: session_1744615678901  ← 新的sessionId
────────────────────────────────
User: "今天想去咖啡馆"
AI: "想要什么样的氛围？"
User: "安静一点"         ← AI记得"咖啡馆"
AI: "推荐悬崖咖啡..."     ← AI记得"咖啡馆+安静"
```

**注意：** Session 2 不会记得 Session 1 的内容（高考失利），因为是不同的对话流。

但是：用户的基础信息（性别、年龄、性格等）会在两个session中都可用，因为存储在 `UserMetadata` 中。

## 总结

### 核心机制

```
每次开启聊天 = 创建新的sessionId
同一个session = 完整的上下文记忆
不同session = 独立的对话主题
所有session = 保存在数据库，可查看历史
```
