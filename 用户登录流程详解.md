## 1. 登录如何保存

### 🔐 认证系统：NextAuth + JWT

**核心配置文件**：`lib/auth.ts`

```typescript
session: {
  strategy: 'jwt',           // 使用JWT策略
  maxAge: 30 * 24 * 60 * 60, // 30天有效期
}
```

### 💾 保存方式

| 存储位置 | 存储内容 | 用途 |
|---------|---------|------|
| **Cookie（HTTP-Only）** | JWT Token | 会话认证，自动携带 |
| **数据库（Prisma）** | 用户完整信息 | 持久化存储 |
| **内存（Session）** | 当前用户状态 | 前端访问 |

### 🔑 JWT Token包含的信息

```typescript
// lib/auth.ts - jwt callback (138-146行)
async jwt({ token, user, account }) {
  if (user) {
    token.id = user.id           // 用户ID
    token.email = user.email     // 邮箱
    token.name = user.name       // 姓名
    token.picture = user.image   // 头像
  }
  return token
}
```

### 🗄️ 数据库存储

**Prisma Schema**：
```prisma
model User {
  id          String    @id @default(cuid())
  email       String?   @unique
  phone       String?   @unique
  name        String?
  password    String?
  gender      String?
  birthDate   String?   // JSON格式：{year, month, day}
  
  // 关联数据
  metadata    UserMetadata?
  chatSessions ChatSession[]
  
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}
```

## 2. 如何判断新用户

### 🆕 新用户判断标准

**核心函数**：`lib/authBridge.ts` - `initializeUserSession()`

```typescript
// 检查birthDate是否为有效的JSON格式
let hasValidBirthDate = false
if (dbUser.birthDate) {
  try {
    const birthData = JSON.parse(dbUser.birthDate)
    hasValidBirthDate = !!(birthData.year && birthData.month && birthData.day)
  } catch (e) {
    hasValidBirthDate = false
  }
}

return {
  userId: dbUser.id,
  userName: userName,
  hasDetailedInfo: !!(dbUser.gender && hasValidBirthDate),  // 🔥 关键判断
  hasMetadata: !!dbUser.metadata
}
```

### ✅ 老用户 vs 🆕 新用户

| 类型 | gender | birthDate | hasDetailedInfo | 跳转目标 |
|------|--------|-----------|-----------------|----------|
| 🆕 **新用户** | ❌ 空 | ❌ 空 | `false` | `/user-info` |
| 🆕 **新用户** | ✅ 有 | ❌ 空 | `false` | `/user-info` |
| 🆕 **新用户** | ❌ 空 | ✅ 有 | `false` | `/user-info` |
| ✅ **老用户** | ✅ 有 | ✅ 有 | `true` | `/home` |

### 🔍 判断流程

```
用户登录成功
  ↓
调用 /api/user/sync
  ↓
执行 initializeUserSession(session)
  ↓
从数据库查询用户信息
  ↓
检查 gender 和 birthDate
  ↓
返回 hasDetailedInfo 状态
  ↓
前端根据状态决定跳转
```

## 3. 如何跳转home界面

### 🚦 跳转控制机制

#### 方式1：`useAuth` Hook自动跳转

**文件**：`hooks/useAuth.ts` (35-41行)

```typescript
// 如果用户还没有填写详细信息，跳转到用户信息页面
if (!data.result?.hasDetailedInfo) {
  console.log('⚠️ [AUTH] 新用户缺少详细信息，跳转到用户信息页面')
  router.push('/user-info')
} else {
  console.log('✅ [AUTH] 老用户已有详细信息，直接进入主页')
  // 这里不需要手动跳转，因为登录页面会自动跳转到 callbackUrl
}
```

#### 方式2：登录页面的自动跳转

**文件**：`app/auth/signin/page.tsx` (26-33行)

```typescript
// 如果用户已登录，直接跳转
useEffect(() => {
  if (status === 'authenticated' && session) {
    const currentPath = typeof window !== 'undefined' ? window.location.pathname : ''
    if (currentPath !== callbackUrl && !currentPath.includes('/home')) {
      window.location.href = callbackUrl  // 默认是 /home
    }
  }
}, [status, session, callbackUrl])
```

#### 方式3：Middleware保护路由

**文件**：`middleware.ts` (24-34行)

```typescript
// 如果未登录，重定向到登录页
if (!isAuth) {
  let from = req.nextUrl.pathname
  if (req.nextUrl.search) {
    from += req.nextUrl.search
  }

  console.log('🔄 [MIDDLEWARE] 未登录用户访问保护页面，重定向到登录页')
  return NextResponse.redirect(
    new URL(`/auth/signin?callbackUrl=${encodeURIComponent(from)}`, req.url)
  )
}
```

**保护的路由**：
```typescript
matcher: [
  '/home/:path*',      // 主页
  '/chat/:path*',      // 聊天
  '/chat-new/:path*',  // 新聊天
  '/history/:path*',   // 历史记录
  '/user-info/:path*', // 用户信息
  '/profile/:path*'    // 个人资料
]
```

#### 方式4：NextAuth回调配置

**文件**：`lib/auth.ts` (130-135行)

```typescript
pages: {
  signIn: '/auth/signin',
  signOut: '/auth/signout',
  error: '/auth/error',
  newUser: '/user-info',  // 🔥 新用户注册后跳转到信息收集页面
}
```

## 4. 完整流程图

### 🎯 登录成功后的跳转流程

```mermaid
graph TD
    A[用户点击登录] --> B{登录成功?}
    B -->|失败| C[显示错误信息]
    B -->|成功| D[NextAuth设置Cookie]
    
    D --> E[前端刷新Session]
    E --> F[useAuth Hook触发]
    F --> G[调用 /api/user/sync]
    
    G --> H[执行 initializeUserSession]
    H --> I[从Prisma查询用户信息]
    
    I --> J{检查 gender & birthDate}
    
    J -->|❌ 缺失| K[hasDetailedInfo = false]
    J -->|✅ 完整| L[hasDetailedInfo = true]
    
    K --> M[router.push('/user-info')]
    M --> N[显示用户信息收集页面]
    N --> O[用户填写信息]
    O --> P[保存到数据库]
    P --> Q[跳转到 /home]
    
    L --> R{当前在登录页?}
    R -->|是| S[window.location.href = '/home']
    R -->|否| T[保持当前页面]
    
    S --> U[显示主页]
    T --> U
    Q --> U
```

### 🔄 登录→主页的详细流程

```
1. 用户输入账号密码
   ↓
2. 调用 signIn('credentials', {...})
   ↓
3. NextAuth验证（lib/auth.ts - authorize函数）
   ├─ 查询数据库
   ├─ 验证密码（bcrypt.compare）
   └─ 返回用户对象
   ↓
4. 生成JWT Token
   ├─ 包含 id, email, name, picture
   └─ 存储在HTTP-Only Cookie中
   ↓
5. 前端刷新Session（update()）
   ↓
6. useAuth Hook监听到登录状态
   ↓
7. 调用 /api/user/sync
   ├─ 执行 initializeUserSession(session)
   ├─ 从Prisma查询用户完整信息
   └─ 返回 hasDetailedInfo 状态
   ↓
8. 根据状态决定跳转
   ├─ hasDetailedInfo = false → /user-info（新用户）
   └─ hasDetailedInfo = true → /home（老用户）
```

## 6. 实际使用示例

### 📝 在组件中使用认证

```typescript
'use client'

import { useAuth } from '@/hooks/useAuth'

export default function MyPage() {
  const { session, isAuthenticated, isLoading, user } = useAuth()
  
  if (isLoading) {
    return <div>加载中...</div>
  }
  
  if (!isAuthenticated) {
    return <div>请先登录</div>
  }
  
  return (
    <div>
      <h1>欢迎，{user?.name}</h1>
      <p>邮箱：{user?.email}</p>
    </div>
  )
}
```

### 🔒 在API中验证登录

```typescript
import { getServerSession } from 'next-auth'
import { authOptions } from '@/lib/auth'

export async function POST(request: Request) {
  // 1. 获取session
  const session = await getServerSession(authOptions)
  
  // 2. 验证登录状态
  if (!session?.user?.email) {
    return NextResponse.json(
      { error: '未登录' },
      { status: 401 }
    )
  }
  
  // 3. 从数据库获取完整用户信息
  const user = await prisma.user.findUnique({
    where: { email: session.user.email }
  })
  
  // 4. 处理业务逻辑
  // ...
}
```

### 🐛 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 登录后立即跳回登录页 | JWT配置错误 | 检查 `NEXTAUTH_SECRET` 环境变量 |
| 登录后卡在 /user-info | birthDate格式错误 | 确保格式为 `{"year":1990,"month":1,"day":1}` |
| Middleware不生效 | 路由匹配错误 | 检查 `middleware.ts` 的 matcher 配置 |
| Session一直为null | Cookie设置失败 | 检查 `NEXTAUTH_URL` 环境变量 |

## 10. 总结

### ✅ 登录流程核心要点

1. **保存方式**：
   - JWT Token存储在HTTP-Only Cookie中（安全）
   - 完整用户信息存储在Prisma数据库中（持久化）

2. **新用户判断**：
   - 检查 `gender` 和 `birthDate` 是否完整
   - `hasDetailedInfo = false` → 新用户

3. **跳转逻辑**：
   - 新用户 → `/user-info`（收集信息）
   - 老用户 → `/home`（直接进入主页）
   - 未登录用户访问保护页面 → `/auth/signin`（Middleware拦截）

4. **自动化流程**：
   - `useAuth` Hook自动处理登录后的跳转
   - Middleware自动保护需要登录的页面
   - Session过期后自动重定向到登录页
