# 重复问题修复

## 问题描述

用户场景：
```
元键入："我昨天晚上想到了一个产品的idea 很开心 然后早上醒得早早早来了公司"

Q1: "公司在哪个区域？周边环境是什么样的？"
A1: "公司在上海市淞虹路 楼下有商圈 早上来公司时还没人来 我还在门外等了一会"

Q2: "公司在哪个区域？早上来公司时周围环境怎么样？" ❌ 重复！
Q3: "公司具体在哪个区域？周边环境有什么特点？" ❌❌ 又重复！
```

**问题：** AI重复问了3次"公司在哪个区域"和"周边环境"

## 修复内容

### 1. ✅ 添加地点位置检测

```typescript
// 🏢 地点位置检测（最重要！）
if (allAnswers.includes('路') || 
    allAnswers.includes('区') || 
    allAnswers.includes('街') || 
    allAnswers.includes('公司在') || 
    allAnswers.includes('淞虹路')) {
  forbidden.push('❌❌❌ 用户已回答地点位置 → 禁止再问：公司在哪、在哪个区域、具体在哪、公司位置')
}
```

### 2. ✅ 添加周边环境检测

```typescript
// 🏢 周边环境检测（最重要！）
if (allAnswers.includes('商圈') || 
    allAnswers.includes('楼下') || 
    allAnswers.includes('周边') || 
    allAnswers.includes('环境')) {
  forbidden.push('❌❌❌ 用户已回答周边环境 → 禁止再问：周边环境怎样、环境特点')
}
```

### 3. ✅ 添加强制检查清单

在生成问题前必须检查：

```
1. ✅ 地点检查：用户是否已回答地点位置？
   - 如果已回答 → 不能再问在哪、哪个区域
   
2. ✅ 环境检查：用户是否已回答周边环境？
   - 如果已回答 → 不能再问环境怎样、有什么
   
3. ✅ 信息覆盖：元键入中是否有多个事件？
   - 如果有 → 不要只问一个事件
   
4. ✅ 问题相似度：和之前的问题是否相似？
   - 如果相似 → 返回[]

如果任何一条未通过 → 返回 {"questions": []}
```

### 4. ✅ 添加具体案例

```
**反面案例（今天的错误）：**
用户说："公司在淞虹路 楼下有商圈"
    ↓
AI还问："公司在哪个区域？周边环境怎样？" ❌ 死刑！
    ↓
应该：检测到已回答 → 返回 {"questions": []} ✅
```

## 现在的逻辑

### 检测流程

```
生成问题前
    ↓
检查用户之前的回答
    ↓
┌────────────────────────────┐
│ 1. 是否包含地点？           │
│    路/区/街/公司在         │
│    → 标记"地点已回答"      │
├────────────────────────────┤
│ 2. 是否包含环境？           │
│    商圈/楼下/周边/环境     │
│    → 标记"环境已回答"      │
├────────────────────────────┤
│ 3. 是否包含其他信息？       │
│    食物/设备/服装/时间     │
│    → 标记相应类别          │
└────────────────────────────┘
    ↓
如果标记了"地点已回答"
    ↓
新问题包含"在哪/区域/位置"？
    ↓
是 → 返回 {"questions": []}  ✅ 正确！
否 → 继续生成问题
```

## 检测规则总结

| 已回答内容 | 触发词 | 禁止再问 |
|-----------|-------|---------|
| 地点位置 | 路/区/街/公司在 | 在哪、哪个区域、具体在哪、公司位置 |
| 周边环境 | 商圈/楼下/周边/环境 | 环境怎样、周围有什么、环境特点 |
| 在家 | 在家/卧室/客厅 | 在哪里、什么地方 |
| 设备 | iPhone/手机/电脑 | 用什么设备 |
| 食物 | 蟹粉/小笼/菜名 | 吃了什么、点了什么 |
| 服装 | 白T恤/裤子/衣服 | 穿什么 |
