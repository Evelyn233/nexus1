# 历史记录图片显示修复说明

## 🐛 问题描述

**用户反馈**：
1. 聊天记录显示"0 张图片"
2. 历史详情页只显示基础场景的图片

**实际情况**：
- 生成了11张图片（3张基础 + 1张心理剧 + 3张假想 + 4张观点）
- 但历史记录只显示3张或0张

## 🔍 问题分析

### 问题1：图片数量为0

**根本原因**：额外场景的图片没有被收集到 `generatedImagesData`

✅ **已修复**（详见 `聊天历史图片数量修复说明.md`）

### 问题2：只显示基础场景图片

**根本原因**：数据格式不匹配

#### 前端保存的格式（修复前）
```typescript
{
  imageUrl: string;
  story: string;
  sceneTitle: string;
}
```

#### 历史详情页期望的格式
```typescript
interface ContentDetail {
  images: Array<{
    sceneTitle: string;
    sceneIndex: number;   // ❌ 缺失
    prompt: string;       // ❌ 缺失
    imageUrl: string;
  }>
}
```

**字段不匹配**：
- ❌ 缺少 `sceneIndex`（场景索引）
- ❌ 缺少 `prompt`（图片描述）
- ✅ 有多余的 `story`（但不影响，历史页不使用）

## 📝 字段说明

### sceneIndex（场景索引）

**作用**：标识图片在生成序列中的位置

**计算方式**：
- **基础场景**：直接使用循环索引 `i`
- **额外场景**：使用 `prev.length`（当前数组长度）

**示例**：
```typescript
// 基础场景
sceneIndex: 0, 1, 2

// 额外场景（心理剧在3张基础场景之后）
sceneIndex: prev.length  // 自动计算为 3, 4, 5, ...
```

### prompt（图片描述）

**作用**：图片的文字描述，用于历史详情页显示

**来源**：
- **基础场景**：`scene.storyFragment`
- **额外场景**：`sceneStory`（从 `scene.storyFragment` 获取）

**显示位置**：
历史详情页的图片下方灰色框中

```tsx
{image.prompt && (
  <div className="text-xs text-gray-600 bg-gray-50 rounded-lg px-3 py-2 mt-2">
    <p className="leading-relaxed">{image.prompt}</p>
  </div>
)}
```
