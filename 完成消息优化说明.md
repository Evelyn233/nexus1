# 完成消息优化说明

## 🐛 问题描述

**用户反馈**：完成消息都是一样的，应该区分不同类型的场景。

**症状**：
```
✨ 创作完成！共生成 3 张图片（包含观点场景和心理剧场景）
✨ 创作完成！共生成 1 张图片（包含观点场景和心理剧场景）
✨ 创作完成！共生成 3 张图片（包含观点场景和心理剧场景）
```

- 多个相同格式的完成消息
- 没有区分场景类型（心理剧、假想场景、观点场景）
- 消息内容不够具体

## 🔍 问题分析

### 1. 根本原因

**多次触发完成消息**：

额外场景生成分3批调用 `generateImagesForNewScenes`：
1. **第2批**（1628行）：心理剧场景
2. **第3批**（1660行）：假想场景
3. **第4批**（1665行）：观点场景

每次调用都会触发 `generateImagesForNewScenes` 函数内的完成消息逻辑（1317行），导致显示3次。

### 2. 为什么会重复？

```
第2批调用 → generateImagesForNewScenes → 完成消息1 ❌
第3批调用 → generateImagesForNewScenes → 完成消息2 ❌
第4批调用 → generateImagesForNewScenes → 完成消息3 ❌
```

每个批次独立执行，互不知道对方的存在，所以都显示了自己的完成消息。

### 3. 为什么消息内容不具体？

旧的完成消息格式：
```typescript
`✨ 创作完成！共生成 ${allScenes.length} 张图片（包含观点场景和心理剧场景）`
```

问题：
- 只显示数量，不区分类型
- 固定文本"包含观点场景和心理剧场景"，不准确
- 每批次只看到自己的场景，看不到全局

## ⏱️ 延迟显示策略

### 为什么需要延迟？

`generateImagesForNewScenes` 是异步调用（fire and forget），不等待完成。如果立即显示完成消息，图片可能还在生成中。

### 延迟时间选择

```typescript
setTimeout(() => {
  // 显示完成消息
}, 3000)  // 延迟3秒
```

**3秒的理由**：
- 图片生成通常需要2-5秒
- 3秒是一个合理的中间值
- 既不会太早（图片还没生成），也不会太晚（用户等待）

### 更好的方案（未来优化）

```typescript
// 监听所有图片生成完成事件
Promise.all(allImagePromises).then(() => {
  // 显示完成消息
})
```

这需要重构 `generateImagesForNewScenes` 为返回Promise的函数。

## 🏗️ 架构说明

### 场景生成流程

```
用户输入
  ↓
基础场景生成（3张，fire and forget）
  ↓
额外场景检测（并行）
  ├─ 心理剧检测 → 生成1张 → generateImagesForNewScenes
  ├─ 假想场景检测 → 生成3张 → generateImagesForNewScenes
  └─ 观点检测 → 生成4张 → generateImagesForNewScenes
  ↓
统一完成消息（延迟3秒）
```
